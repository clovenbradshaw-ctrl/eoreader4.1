Hard coreference / identity cases, run through the real deterministic core — admission gravity, union-find identity, the symbolic conflict oracle, the DEF·EVA·REC ledger — the same src/ modules the app loads, with no model, no embedder, no weights, no network. The panel places each on the one axis that matters: does resolving it need the WITNESS CHANNEL to read meaning? The tempting cut — hand-coded vs learned — is wrong, because the engine’s middle tier already holds a corpus-learned statistical layer (Fellegi-Sunter m/u weights, discriminativeness, the REC ledger). Learned is not the model. The boundary is meaning: a witness reads it and deposits defeasible weight into the field; it never decides (coref.js: “a witness observes, it does not decide”). Most cases sit at ENGINE (the next deterministic build) or MIXED (a learned core, a witness tail); only a few genuinely cross to MODEL.
